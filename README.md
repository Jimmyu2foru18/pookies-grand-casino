## Pookie's Grand Casino

A comprehensive browser-based card game suite featuring Solitaire, Blackjack, 5‑Card Draw Poker, Texas Hold'em, and Gin Rummy. The experience uses virtual chips only and is designed as a safe, simulated gambling environment.

### Tech Stack

- React 18 with TypeScript
- Vite as the build tool and dev server
- Tailwind CSS via CDN for styling
- @dnd-kit for drag-and-drop interactions (Solitaire and Rummy)
- lucide-react for icons

### Features

- **Multiple Game Modes**
  - **Solitaire (Klondike, Vegas-style)** with a buy-in and per-card rewards.
  - **Blackjack** vs. a dealer (Sachi) with standard 21 rules.
  - **5‑Card Draw Poker** with bots, ante, draw, and showdown.
  - **Texas Hold'em** with multi-stage betting (Pre-Flop, Flop, Turn, River).
  - **Gin Rummy** style play with sets, runs, draw, and discard.
- **Virtual Chip Economy**
  - Start with an initial chip balance.
  - Bets, wins, and losses update your chip count across games.
- **Bot Players**
  - Named bot opponents with their own chip stacks and actions.
- **Polished UI**
  - Themed backgrounds for each table.
  - Pookie avatar and casino‑style overlays.
  - In‑game rules modal and help text per game.

### Safety & Disclaimer

This project is a **gambling simulation for entertainment only**:

- No real money is wagered, won, or lost.

### Getting Started (Local Development)

Requirements:

- Node.js 18+ and npm

Install dependencies:

```bash
npm install
```

Run the development server:

```bash
npm run dev
```

Then open the printed local URL (typically `http://localhost:5173/`) in your browser.

Build for production:

```bash
npm run build
```

Preview the production build locally:

```bash
npm run preview
```

### Project Structure

- `App.tsx` – Top‑level application shell, menu, game selection, and layout.
- `components/GameBoard.tsx` – Core game engine handling all game modes and table logic.
- `components/Card.tsx` – Card rendering and interaction.
- `services/cardLogic.ts` – Deck creation, shuffling, and card utility logic.
- `constants.ts` – Game configuration, backgrounds, winnings, and rules text.
- `types.ts` – Shared TypeScript types and enums.
- `public/` – Static assets such as backgrounds, avatars, and card faces.
- `dist/` – Build output generated by `npm run build` (not edited by hand).
